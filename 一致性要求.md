# 每一局里应始终保持一致的内容

下面这些字段，在**同一局游戏的整条链路中建议保持一致**：

## 必须始终一致

### 1. session

```
"session": {
  "session_id": "sess_consistency_001",
  "player_count": 1,
  "difficulty": "NORMAL"
}
```

### 2. seed

```
"seed": {
  "run_seed": "run_consistency_001"
}
```

### 3. constraints

```
"constraints": {
  "language": "cn",
  "content_rating": "PG",
  "max_chars_scene": 220,
  "max_chars_option": 14,
  "forbidden_terms": ["骰子", "扑克", "点数", "规则"]
}
```

### 4. slots

```
"slots": {
  "tone_bias": "紧张冒险",
  "theme_bias": "大山寻宝",
  "npc_bias": "山神"
}
```

### 5. client_context

```
"client_context": {
  "platform": "web",
  "locale": "zh-CN"
}
```

------

## 应保持逻辑连续，但数值会变化

### 1. time

这一组要**连续推进**：

- `hard_limit_seconds` 固定为 300
- `elapsed_active_seconds` 逐步增加
- `remaining_seconds = 300 - elapsed_active_seconds`

### 2. context.current_scene_summary

必须承接上一步输出的场景摘要。

### 3. context.available_options

必须承接上一步输出的 options。

### 4. payload.selected_option_id

必须来自上一步 `available_options` 中真实存在的 id。

### 5. context.state_flags

可以逐步累积，例如：

- `cliff_runes_discovered`
- `guardian_alerted`
- `fire_ring_activated`
- `core_gate_opened`

### 6. END 的 history_events

必须是前面事件链的浓缩汇总，不能脱节。`END` 样例里也是这样设计的。
