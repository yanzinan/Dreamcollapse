# 一、游戏生命周期（Game Lifecycle）
本游戏为 5 分钟内完成的一局 AI 驱动叙事游戏。

游戏逻辑结构为：

> **INIT → LOOP → END**
>

生命周期与事件彻底分层：

+ Stage = 游戏阶段
+ Event = 阶段下发生的具体事件

---

# 二、Stage 1：INIT（起始阶段）
### 作用
+ 建立故事主线
+ 确定世界设定
+ 告知玩家身份
+ 给出第一步行动方向

INIT 阶段只发生一次。

---

## INIT 阶段下的事件
### 🟢 INIT 事件
**说明：**

+ 游戏开局时调用
+ 返回故事框架
+ 生成世界基调
+ 引导进入 LOOP 阶段

INIT 事件完成后，自动进入 LOOP 阶段。

---

# 三、Stage 2：LOOP（核心循环阶段）
这是整局游戏的主体阶段。

在 LOOP 阶段内：

+ AI 根据当前活跃时间
+ 根据游戏难度
+ 根据玩家反馈
+ 决定下一事件类型

LOOP 会持续循环，直到 AI 决定进入 END 阶段。

---

## LOOP 阶段下的事件类型
---

### 🔵 DECISION 事件
**作用：**

+ 提供剧情选择
+ 承接上一个结果
+ 决定下一个事件方向

功能定位：

+ 连接器
+ 分支生成器
+ 节奏调节器

---

### 🔴 COMBAT 事件
**作用：**

+ 进行战斗或风险行动结算
+ 通过扑克机制进行对抗
+ 输出胜负或混合结果

功能定位：

+ 紧张制造
+ 难度体现
+ 状态变化节点

---

### 🟡 PUZZLE 事件
**作用：**

+ 提供逻辑或解谜挑战
+ 玩家输入答案
+ AI 判定结果

功能定位：

+ 脑力挑战
+ 节奏变化
+ 非战斗型风险处理

---

# 四、Stage 3：END（终结阶段）
当 AI 决定本局结束时进入 END 阶段。

触发条件示例：

+ 时间接近耗尽
+ 剧情完成
+ 玩家胜利或失败

END 阶段不再回到 LOOP。

---

## END 阶段下的事件
---

### 🔵 END 事件（结算事件）
**作用：**

+ 输出游戏结果
+ 生成结算数值
+ 说明胜负原因
+ 返回成就或评分

用于客户端展示结算界面。

---

### 📘 NOVEL 事件（微小说输出事件）
**作用：**

+ 将本局经历整合为短篇故事
+ 文学化输出
+ 提供分享与情绪价值

特点：

+ 不影响胜负
+ 仅用于内容输出
+ 属于 END 阶段附加事件

---

# 生命周期结构示意
Stage: INIT  
  Event: INIT

Stage: LOOP  
  Event: DECISION  
  Event: COMBAT  
  Event: PUZZLE

Stage: END  
  Event: END  
  Event: NOVEL

---

这样现在：

+ 阶段与事件彻底分离
+ 逻辑表达非常清晰
+ 讲给合伙人听就是：INIT → LOOP → END
+ 不会出现“起始是决策还是阶段”的混乱

这版结构是健康的。
